# 位置微调：缩放、改宽高与吸附设计

## 目标

HTML Mender 的“位置微调”用于解决 HTML PPT 最后一轮排版问题：

- 元素差一点没对齐。
- 标题、图片或卡片需要微移。
- 元素需要稍微放大、缩小、拉宽或压高。

本阶段不做 PDF 导出，优先补齐尺寸微调和后续吸附能力。

## 产品原则

- 默认安全：`移动缩放` 使用 `transform`，尽量不重排页面。
- 明确选择：`改宽高` 会写真实 `width` / `height`，用户主动选择后再使用。
- 可恢复：所有位置、缩放、尺寸调整都能撤销，也能一键恢复。
- 干净导出：只把必要样式 patch 回源 HTML，不保存编辑器 UI 和运行时状态。

## 工具条

进入 `位置微调` 并选中元素后，浮层显示：

```text
移动缩放 | 改宽高 | 恢复
```

### 移动缩放

- 拖动外框：移动元素。
- 拖四角控制点：整体视觉缩放。
- 使用 `transform: translate(...) scale(...)`。
- 适合“看起来再大一点、再小一点、挪一点”。

### 改宽高

- 左 / 右控制点：只改宽。
- 上 / 下控制点：只改高。
- 四角控制点：同时改宽高。
- 直接写入 inline `width` / `height`。
- 适合“文本框变宽”“图片框变扁”“卡片拉伸”。

### 恢复

清除编辑器追加的：

- `--hsm-layout-x`
- `--hsm-layout-y`
- `--hsm-layout-scale`
- `--hsm-layout-width`
- `--hsm-layout-height`
- `transform` 中的移动和缩放
- 编辑器写入的真实 `width` / `height`

同时保留元素原始 transform 和原始内联宽高。

## 操作反馈

选中框标签显示当前调整：

```text
位置 · x +12 / y -8 · 108%
位置 · w 420 / h 180
```

缩放范围：

- 最小 `20%`
- 最大 `500%`

改宽高范围：

- 最小 `8px`
- 最大 `8000px`

## 交互规则

- 拖动超过 `3px` 才记录为一次修改。
- 松手后写入历史栈。
- `Esc` 取消当前拖拽。
- 方向键移动选中元素：普通 `1px`，`Shift` 为 `10px`。
- 左边 / 上边改宽高时追加位移，让拖动边尽量跟手。

## 技术方案

状态模型：

```js
{
  x: 0,
  y: 0,
  scale: 1,
  width: null,
  height: null,
  baseTransform: "",
  baseTransformOrigin: "",
  baseWidthStyle: "",
  baseHeightStyle: ""
}
```

`移动缩放` 合成 transform：

```js
[
  baseTransform,
  `translate(${x}px, ${y}px)`,
  `scale(${scale})`
].filter(Boolean).join(" ");
```

`改宽高` 写入：

```css
width: 420px;
height: 180px;
```

首次改宽高时记录原始内联宽高，用于恢复。

## 对齐参考线 / 吸附

后续做移动吸附，先不强绑缩放吸附。

参考线：

- 元素 left / centerX / right
- 元素 top / centerY / bottom
- 当前视口水平中心和垂直中心

规则：

- 阈值 `6px`
- 按住 `Alt` 临时关闭吸附
- 排除当前拖动元素、编辑器 UI 和大面积背景层

## 验收用例

1. 进入 `位置微调`，拖动元素，导出后位置保留。
2. 使用 `移动缩放` 拖四角，元素视觉放大或缩小，周围布局不被挤开。
3. 使用 `改宽高` 拖左右边，只改变宽度。
4. 使用 `改宽高` 拖上下边，只改变高度。
5. 点击 `恢复` 后，移动、缩放、宽高修改都被清除。
6. 已有 transform 或 inline 宽高的元素，恢复后保留原始值。
7. 下载 HTML 后不包含编辑器 UI，版面调整保留。
