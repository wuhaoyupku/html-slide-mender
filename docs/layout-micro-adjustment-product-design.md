# HTML Slide Mender 版面微调产品设计稿

## 1. 背景

HTML Slide Mender 现有能力主要解决文字修稿和图片替换问题。但真实用户在编辑 AI 生成的 HTML PPT 时，经常遇到的问题不是内容错误，而是版面差一点：

- 标题需要再往上或往左一点。
- 图片需要再大一点，或者和旁边文字对齐。
- 卡片之间间距不一致。
- 一组元素没有居中。
- 文本框太窄，改字后显得拥挤。
- AI 生成的版面整体可用，但局部位置不舒服。

这类需求不适合重新生成整页，也不值得让用户改 HTML/CSS。它需要的是一种介于“内容编辑”和“完整设计工具”之间的轻量能力。

一句话定义：

> 在不重排页面、不接管幻灯片结构的前提下，让用户对当前页元素做安全的视觉位置和尺寸微调。

## 2. 产品目标

### 2.1 解决什么

- 让用户可以拖动当前页里的文字、图片、卡片、图标等元素。
- 让用户可以用方向键做 1px 级微调。
- 让用户可以调整部分元素的宽高。
- 让用户可以看到对齐参考线，快速对齐元素。
- 让用户可以撤销、重做、恢复原位置。
- 让导出的 HTML 保留微调结果，但不保留插件浮层。

### 2.2 不解决什么

第一版不做完整排版系统：

- 不新增页面。
- 不删除页面。
- 不重排 slide 结构。
- 不把普通流式布局强行改成绝对定位。
- 不做完整图层面板。
- 不做复杂 flex/grid 编辑器。
- 不做跨页母版。
- 不保证可以编辑所有网站布局。

## 3. 核心原则

### 3.1 只做视觉微调

默认通过 `transform: translate(...)` 和少量 inline style 记录用户调整。

优先避免修改会影响页面布局流的属性，例如：

- `margin`
- `padding`
- `display`
- `position`
- `grid-template-*`
- `flex`

### 3.2 不挤压周围元素

用户拖动一个元素时，周围元素不应被推开。

这意味着第一版移动操作优先使用视觉偏移，而不是改变 DOM 顺序或布局规则。

### 3.3 当前页内操作

插件仍然不接管原页面导航。

用户翻到哪一页，就微调哪一页。扫描逻辑仍以当前可见区域为准。

### 3.4 可恢复

所有版面微调都必须支持：

- 撤销
- 重做
- 恢复当前元素原始位置
- 恢复当前元素原始尺寸

### 3.5 风险分级

不同操作对版面的影响不同，产品上要分成两类。

安全操作默认开放：

- 拖动视觉位置
- 方向键微调
- 吸附对齐
- 恢复原位置
- 图片框等明确容器的等比缩放

风险操作需要提示：

- 修改普通文本流元素宽高
- 修改 flex/grid 子项尺寸
- 修改大容器尺寸
- 修改会导致文字溢出的尺寸

## 4. 目标用户场景

### 场景 A：标题位置不舒服

1. 用户打开 HTML PPT。
2. 点击插件进入编辑模式。
3. 选中当前页标题。
4. 拖动标题向上移动 12px。
5. 插件显示参考线，提示标题已和左侧图形顶部对齐。
6. 用户点击下载 HTML。

### 场景 B：图片需要放大并对齐

1. 用户选中图片外框。
2. 拖动右下角控制点放大图片框。
3. 图片内容保持原有裁切规则。
4. 用户拖动图片框，使其右边缘和标题右边缘吸附。
5. 插件记录位置和尺寸修改。

### 场景 C：几个卡片间距不一致

1. 用户依次选中三个卡片。
2. 插件进入多选状态。
3. 用户点击“水平等距”。
4. 插件只调整卡片视觉偏移，不改变原 DOM 结构。
5. 用户保存。

### 场景 D：改字后文本框太窄

1. 用户修改标题文案。
2. 插件提示文字可能溢出。
3. 用户拖宽文本框。
4. 插件检测溢出消失。
5. 保存前检查通过。

## 5. 功能范围

### 5.1 V1 必须有

- 单元素拖动位置。
- 方向键微调。
- `Shift + 方向键` 快速移动。
- 拖动时显示当前偏移量。
- 拖动时限制元素不要完全移出当前视口。
- 撤销 / 重做。
- 恢复当前元素位置。
- 导出时保留位置修改。

### 5.2 V1 应该有

- 拖动时显示对齐参考线。
- 边缘吸附和中心线吸附。
- 支持图片、文本、卡片类块元素。
- 区分“图片内容调整”和“图片框调整”。
- 保存前统计版面修改数量。

### 5.3 V1 可以后置

- 多选。
- 等距分布。
- 宽高调整。
- 锁定比例缩放。
- 手动输入 X/Y/W/H。
- 网格吸附。
- 跨页统一对齐。

### 5.4 明确不做

- 不做完整画布。
- 不做左侧图层树。
- 不做自动重排。
- 不做复杂约束布局。
- 不把页面转成自定义 JSON 再渲染。

## 6. 交互设计

### 6.1 进入版面微调模式

现有顶部工具栏增加一个模式切换：

```text
文本 | 图片 | 版面 | 撤销 | 重做 | 显示框 | 下载 | 退出
```

选择“版面”后：

- 文本不直接进入输入状态。
- 图片不直接进入替换状态。
- 点击元素选中外框。
- 拖动外框即可移动元素。

### 6.2 元素选框

版面模式下，所有可微调元素使用统一蓝紫色边框。

选中元素后显示：

- 四角控制点。
- 当前偏移量标签，例如 `x +12 / y -8`。
- 小型浮动工具条。

浮动工具条：

```text
恢复位置 | 置于前方 | 置于后方 | 更多
```

V1 可以先只实现：

```text
恢复位置
```

### 6.3 拖动移动

拖动规则：

- 鼠标按下选框后开始拖动。
- 拖动超过 3px 才记录为移动，避免误触。
- 拖动过程中实时应用视觉偏移。
- 松手后写入历史栈。
- 按 `Esc` 取消本次拖动。

快捷键：

- `ArrowUp`：上移 1px。
- `ArrowDown`：下移 1px。
- `ArrowLeft`：左移 1px。
- `ArrowRight`：右移 1px。
- `Shift + Arrow`：移动 10px。
- `Cmd/Ctrl + Z`：撤销。
- `Cmd/Ctrl + Shift + Z`：重做。

### 6.4 吸附参考线

拖动时计算当前元素与其他可见元素的位置关系。

参考线类型：

- 左边缘对齐。
- 右边缘对齐。
- 顶部对齐。
- 底部对齐。
- 水平中心对齐。
- 垂直中心对齐。

吸附阈值：

- 默认 6px。
- 用户拖动到阈值内时自动吸附。
- 按住 `Alt` 时临时关闭吸附。

### 6.5 尺寸调整

V1 可以先隐藏尺寸控制点，只做位置微调。

如果进入 V1.1，尺寸调整规则如下：

- 明确图片框、卡片、按钮、标题块可以调整宽高。
- 普通段落、列表、flex/grid 子项调整前给出风险提示。
- 调整文本框尺寸后立即检测文字溢出。
- 图片框调整尺寸不等于图片内容缩放，二者在 UI 上要区分。

图片相关 UI 分两层：

```text
调整图片内容：填满 / 适应 / 放大 / 缩小 / 拖动画面
调整图片框：移动 / 改宽高 / 对齐
```

## 7. 元素识别规则

### 7.1 可移动元素

优先识别：

- 已识别的文本元素。
- 已识别的图片元素。
- 带 `data-editable` 的元素。
- 带 `data-layout-editable` 的元素。
- 带 `data-card`、`data-panel`、`data-shape` 语义标记的元素。
- 有可见边界、背景、边框或阴影的块级容器。

### 7.2 不建议移动的元素

默认不显示移动框，或标记为高风险：

- `body`
- `html`
- 当前 slide 根容器
- 全屏背景层
- 过大的容器，面积超过视口 70%
- 导航按钮
- 插件 UI
- `script`、`style`、`canvas`、`iframe`

### 7.3 风险判断

元素满足以下条件时，标记为“安全可移动”：

- 在当前视口内可见。
- 宽高大于最小阈值。
- 面积小于视口 70%。
- 不是 slide 根容器。
- 不是页面导航控件。

元素满足以下条件时，标记为“谨慎调整”：

- `position: static` 且处于普通文档流。
- 父级是 flex/grid 容器。
- 元素宽高由内容撑开。
- 元素包含多个复杂子元素。

谨慎调整元素仍可用 `transform` 移动，但尺寸调整默认关闭。

## 8. 技术实现方案

### 8.1 状态模型

新增布局调整状态：

```js
layoutAdjustments = new Map([
  [itemId, {
    x: 0,
    y: 0,
    width: null,
    height: null,
    baseTransform: "",
    baseWidth: "",
    baseHeight: "",
    risk: "safe"
  }]
]);
```

### 8.2 移动实现

优先记录原有 transform，再由 JS 生成完整 transform 字符串，避免出现无效的 `none translate(...)`：

```js
const base = adjustment.baseTransform && adjustment.baseTransform !== "none"
  ? adjustment.baseTransform
  : "";
element.style.transform = `${base} translate(${adjustment.x}px, ${adjustment.y}px)`.trim();
```

如果原元素已有 transform：

- 第一次移动时记录到 `baseTransform`。
- 后续只更新 `x` 和 `y`。

如果原 transform 是 `none`：

- `baseTransform` 为空。
- 只追加 translate。

### 8.3 尺寸实现

V1 先不做默认尺寸调整。

V1.1 支持后，优先写入：

```css
width: <value>px;
height: <value>px;
```

对于图片框：

- 锁定 frame 宽高。
- 图片内容仍使用现有图片调整逻辑。

对于文本框：

- 改宽高后检测 `scrollWidth > clientWidth` 或 `scrollHeight > clientHeight`。
- 如果溢出，选框变橙色。

### 8.4 导出方案

沿用现有内容层 patch 思路。

导出时需要支持新的修改类型：

- `layout`
- `layout-size`

对于 source-based export：

- 找到原始元素 selector。
- 回写元素 `style`。
- 只回写用户实际修改过的元素。

对于 full export：

- clone DOM 后移除插件 UI。
- 保留元素 inline style 中的布局微调属性。

### 8.5 历史记录

拖动开始时捕获 before state。

拖动结束时捕获 after state。

只有 before 和 after 不一致时才写入历史：

```text
Move element
Resize element
Reset layout
```

撤销时恢复：

- style
- layout adjustment map
- 选框位置

### 8.6 与现有图片调整的关系

图片现在已有“图片内容拖动”和“缩放”能力。

新增版面微调后要明确区分：

- 图片内容调整：移动图片内部画面，不动外框。
- 图片框调整：移动或改变图片所在元素外框。

交互上：

- 图片模式默认编辑图片内容。
- 版面模式默认编辑图片框。

## 9. 安全保护

### 9.1 出界检测

拖动后如果元素完全离开当前视口，禁止释放到该位置。

如果元素部分出界，显示橙色提示：

```text
元素部分超出当前页面
```

### 9.2 重叠检测

保存前检测可见元素之间的显著重叠。

第一版只做提示，不阻止保存：

```text
有 2 个元素可能重叠，仍然下载？
```

### 9.3 溢出检测

文本元素移动或改尺寸后，继续使用现有文字溢出检测。

如果有溢出：

- 选框变橙色。
- 保存面板提示。

### 9.4 恢复机制

每个被移动过的元素都支持：

- 恢复位置。
- 恢复尺寸。
- 删除当前元素的所有版面微调。

## 10. 保存前检查

下载 HTML 前，保存面板增加版面统计：

```text
已修改：
文字 8 处
图片 2 处
版面 5 处

检查：
文字溢出 1 处
元素出界 0 处
疑似重叠 2 处
```

如果存在风险：

```text
发现可能影响版面的调整

- 1 处文字可能溢出
- 2 处元素可能重叠

[继续下载] [返回修改]
```

## 12. 成功标准

功能成功：

- 用户能在 10 秒内把一个标题移动到满意位置。
- 用户能用方向键做精细调整。
- 用户能撤销误操作。
- 用户能恢复元素原位置。
- 导出的 HTML 能保留移动结果。
- 导出的 HTML 不包含插件 UI。

体验成功：

- 用户理解“这是微调，不是重新排版”。
- 用户不会因为拖动一个元素导致整页大范围错乱。
- 出现风险时，用户能看懂风险来自哪里。

技术成功：

- 现有文字和图片编辑不被破坏。
- 对已有 transform 的元素能安全叠加偏移。
- 动态 HTML PPT 的 source-based export 能正确回写布局 style。

## 13. 迭代路线

### V1：位置微调版

目标：解决“往上一点、往左一点、对齐一点”。

- 新增版面模式。
- 单元素拖动。
- 方向键微调。
- 偏移量显示。
- 恢复位置。
- 撤销 / 重做。
- 导出保留位置修改。

目标：让用户调得更准。

- 边缘参考线。
- 中心参考线。
- 6px 吸附。
- Alt 暂停吸附。
- 保存前出界检查。

### V1.2：尺寸微调版

目标：解决“框太窄、图太小、卡片不齐”。

- 图片框宽高调整。
- 文本框宽度调整。
- 等比缩放。
- 尺寸恢复。
- 文本溢出检测联动。

### V1.3：多选排齐版

目标：解决多个元素之间的位置关系。

- 多选。
- 左 / 中 / 右对齐。
- 顶 / 中 / 底对齐。
- 水平等距。
- 垂直等距。

## 15. 第一版验收用例

### 用例 1：移动标题

1. 打开 sample deck。
2. 进入版面模式。
3. 选中标题。
4. 向上拖动 20px。
5. 下载 HTML。
6. 打开导出文件，标题位置保持移动后状态。

### 用例 2：方向键微调

1. 选中一个文本块。
2. 按右方向键 3 次。
3. 元素右移 3px。
4. 按 `Cmd/Ctrl + Z`。
5. 元素回到原位。

### 用例 3：恢复位置

1. 拖动图片到新位置。
2. 点击恢复位置。
3. 图片回到原始位置。
4. 保存后导出文件无该图片的位置微调。

### 用例 4：不破坏图片内容调整

1. 图片模式下调整图片内容缩放。
2. 版面模式下移动图片框。
3. 下载 HTML。
4. 图片内容缩放和图片框位置都保持正确。

### 用例 5：已有 transform 元素

1. 打开包含 transform 动画或初始位移的 deck。
2. 移动一个已有 transform 的元素。
3. 元素保持原 transform 效果，并叠加用户位移。
4. 撤销后恢复原 transform。
